Если нужно применять рендеринг шаблонов, то нужно контролер должен реализовывать базовый.
Для нашего кода добавим.

            use Symfony\Bundle\FrameworkBundle\Controller\AbstractController;

            class ArticleController extends AbstractController
 делаем для того, чтобы иметь доступ к базовым методам, например
                return $this->render();

     1-й аргумент это имя файла шаблона
     2-м передаем массив переменных для использования в шаблоне.
 Разумеется массив переменных должен браться например из БД (статьи для блога).

Шаблонизатор Twig 3 синтаксиса
 1. {{....}}    в таком можем распечатать что угодно переменную, строку, или сложное выражение.
 2. {{%...%}}   в таком что угодно делать. например if или for
 3. {#.....#}   комментарии


Есть отсылка к документации
    https://twig.symfony.com/

В twig имеются функции, есть тесты
 наиболее полезен раздел Filters.
                    например      |length           // подсчитывает кол-во переданного слева
                            commetns|length
    Результаты действия одного фильтра можно передавать в другие
            {{ name|upper|lower|upper|lower|upper }}

НАСЛЕДОВАНИЕ шаблонов twig
    Это супер фича шаблонов Твиг

Например для получения базовой структуры HTML документа, добавим в наш шаблон - шаблон базовый.
            {% extends 'base.html.twig' %}
