Установим Symfony Security Checker  для девелоп
            composer require sec-checker
Для девелопа будет
            composer require sec-checker --dev
Вообще имя пакета в таком виде    sec-checker    невалидное.

Изначально проект Symfony маленький и включает несколько зависимостей. Одна из которых это
    "symfony/flex": "^1.1",

Flex это плагин для Composer у которого есть 2 суперсилы.
1-я супер сила это система алиасов.
   https://symfony.sh/   на этом сайте все рецепты доступные. И у некоторых из них есть алиасы
       так например для библиотеки sensiolabs/security-checker
              Aliases sec-check sec-checker sec-checks security-check security-checker

    Это супер.
     Нужен логгер
        composer require logger
     Нужно отправлять email?
        composer require mailer

2-я супер сила, еще лучше это рецепты.
    Например при добавлении зависимости выше кроме ожидаемых изменений в файлах composer.json и composer.lock
     был изменен файл               symfony.lock
     и появился новый файл конфига         security_checker.yaml

    Файл    symfony.lock    - файл управления Flex. Он отслеживает и следит, какие рецепты были установлены.
      в основном в нем коммиты в Git.

    Файл    security_checker.yaml   был создан с помощью рецепта. Этот конфиг добавляет в наше приложение новые консольные команды bin/console
     Благодаря этому теперь можно запустить
                php bin/console security:check
         И в качестве вывода увидеть например:
                        Symfony Security Check Report
                        =============================

                        No packages have known vulnerabilities.

   Теперь каждый раз при установке пакетов, Flex   будет выполнять для этого пакета  рецепт (конечно же если рецепт для пакета существует.).

    Рецепты могут добавлять файлы конфиги, создавать каталоги, изменять имеющиеся файлы такие как .gitignore и другие, делается это для достижения той цели, чтобы библиотека мгновенно начинала работать без дополнительных настроек.

Ну а цель установленого ранее пакет security-checker это проверить есть ли в используемых в проекте пакеты, с известными уязвимостями.
 Можно это делать через консольную команду.
  Но вообще, если посмотреть
                git diff composer.json
    то увидим, что появилась секция
            "scripts": {
                    "auto-scripts": {
                        "security-checker security:check": "script"
                    },
     таким образом теперь при выполнении
            composer install
      в конце выполнится проверка   security checker


При этом при удалении пакетов   Flex    будет удалять связанные с ними рецепты.
  Это облегчает тестирование новых пакетов.

Рецепты находятся в репозиториях например в файле manifest.json.
