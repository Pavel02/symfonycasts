Env Var Tricks & on Production

Загрузка переменных сред из .env  влияет на производительность (милисек).
 Для небольших проектов норм, но с ростом рекомендуется подумать о других способах задания переменных окружения.

Если значения переменных сред  из файла  .env  будет использоваться толко для разработки, то следует этот пакет удалить,
 и затем установить как для разработки
            composer remove symfony/dotenv

            composer require symfony/dotenv --dev


Есть ограничение, что перемен среды это всегда Строки.
 И для задания булевых значений приходится указывать   "false"  "true"

 Лайфхак использование префиксов
        nexy_slack:
            endpoint: '%env(string:SLACK_WEBHOOK_ENDPOINT)%'



string:
bool:
float:
resolve:
file:       // контент файла
base64:
constant:   // чтение  PHP  констант
json:


И эти префиксы можно объединять, пример:
        app.secrets: '%env(json:file:SECRETS_FILE)%'
